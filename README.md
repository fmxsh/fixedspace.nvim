# Fixed space

Plugin to create a window to the right in NeoVim, not able to be resized or entered.

Part of a quick solution for [mdtoc.nvim](https://github.com/fmxsh/mdtoc.nvim).

Used together with _mdtoc.nvim_, which is a floating window with a table of contents for markdown files. It is undesired to have the main window text expand beneath the floating window, so this plugin creates a fixed space under the floating table of contents.

Perhaps a normal window could be used for the float for whatever purpose, but this was a simpler solution at the moment to get it up working.

## About the code

The code of this plugin was mainly boilerplate AI generated by ChatGPT 4o.

## Installation

Integrated into my own project manager, to close and open on project switching.

This goes into `.config/nvim/lua/custom/plugins/fixedspace.lua`, which is loaded by my highly modified kickstart.lua running Lazy plugin manager.

> [!Note]
> This is not provided in a user friendly way and not expected to be used as it is.

```lua
return {
  'fmxsh/fixedspace.nvim',
  dependencies = {},
  config = function()
    require('fixedspace').setup {}
    vim.defer_fn(function()
      require('fixedspace').toggle()
    end, 1)

    vim.api.nvim_create_autocmd('User', {
      pattern = 'preSwitchToProject',
      callback = function()
        require('fixedspace').disable()
      end,
    })

    vim.api.nvim_create_autocmd('User', {
      pattern = 'postSwitchToProject',
      callback = function()
        require('fixedspace').enable()
      end,
    })
  end,
}
```
